Interop Index

The Interop Index is a listing of:

  • exchanged by
  • used to exchange by
  • constructed from exchanging through
  • , which are side-channels to

Its purpose is to map out the different potential relations between deliberative tools and develop new workflows.

Please add to this with the tools you create and interoperate with. Please be liberal with linking to other tools, objects, interfaces and workflows. If in doubt, link: we would rather have false positives than negatives.

Domain Objects

A domain object is a kind of data that is exchanged when inter-operating with another deliberative tool. It is an abstract concept, which may be represented by different datatypes.

Please add domain objects. It’s pretty free-form. The purpose is to have something to link to so we can discover systems that share the same domain.

Non-interop interfaces

A non-interop interface is an interface to a deliberative tool that is not used for interoperating with other deliberative tools.

From the perspective of , the non-interop interfaces of a tool are a side-channel.
